¿Cómo funciona este tablero?

Es un mini-sistema de microservicios: Angular en el frontend y un backend reactivo en Spring Boot (WebFlux + R2DBC), pensado para no bloquear hilos mientras espera a la base de datos.

  • Tiempo real sin refrescar: cada vez que alguien crea, mueve o borra una tarea, el backend emite un evento por Server-Sent Events (SSE) a todos los navegadores conectados — por eso lo ves cambiar al instante en cualquier pestaña, incluso desde otro país.
  • Drag & drop persistente: al soltar una tarjeta, el frontend reordena localmente y confirma el nuevo orden con el backend en una sola llamada por lote.
  • Demo público moderado: como no hay login, cada visitante se identifica por IP para limitar cuántas tareas puede crear, y hay un filtro de lenguaje ofensivo con avisos progresivos antes de bloquear.
  • Desplegado gratis: frontend en Vercel, backend (Docker) en Render, y Postgres en Neon.

Código fuente: github.com/BorisHernan/Proyecto_Portafolio
