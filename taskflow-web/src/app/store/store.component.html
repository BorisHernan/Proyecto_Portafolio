¿Cómo funciona esta tienda?

Es un flujo de compra simulado para mostrar cómo se construye un carrito, cupones y una boleta — pero no todo es de mentira:

  • Catálogo, carrito, cupones y boleta: viven en el navegador. No hay pasarela de pago ni tarjetas reales, y no se guarda ninguna "orden" en ningún lado.
  • El stock sí es real: cada compra descuenta unidades de verdad en la base de datos, y lo ves bajar en vivo en cualquier pestaña o dispositivo — el mismo mecanismo de Server-Sent Events del tablero Kanban, aplicado aquí.
  • Cupones fijos: son un puñado de códigos predefinidos (los ves con el botón "¿Qué cupones hay?" en el carrito), no un sistema de descuentos real.
  • Se repone solo: el stock vuelve a sus valores iniciales una vez al día, para que la demo no se agote.

API en vivo: Swagger / OpenAPI
